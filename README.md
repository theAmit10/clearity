# Habit Tracker (offline, React Native CLI, RN 0.86 / New Architecture)

A HabitKit-style habit tracker. 100% offline — no backend, no network calls anywhere in the app.

This is a **real React Native CLI project** — it was generated with the official
`@react-native-community/cli` template, so it includes real `android/` and `ios/`
native folders (Gradle project, Xcode project, Podfile). Nothing here is a stub.

- React Native **0.86.0** (current stable as of this build)
- React **19.2.3**
- **New Architecture only** — as of RN 0.82, the New Architecture (Fabric/TurboModules/JSI)
  is mandatory and can no longer be disabled, so there's no `newArchEnabled` flag to touch.
- **react-native-reanimated 4.x** — v4 requires the New Architecture (which RN 0.86 always
  has) and moved its worklet transform into a separate `react-native-worklets` package.
  Both are already wired up in `package.json` and `babel.config.js`.

## Features implemented
- Create / edit / delete habits (name, emoji icon, color)
- Daily toggle + GitHub-style contribution heatmap per habit (react-native-svg)
- Tap any past day on the detail screen to backfill/unmark it
- Streak stats: current streak, best streak, total completions, animated 30-day ring
- **Data export/import**: exports a JSON backup via the native share sheet (AirDrop, email,
  files, etc.) and imports it back on another device, with merge-or-replace choice
- **Local-only diagnostics**: global error handler + manual event log, stored on-device,
  exportable the same way as habit data. No analytics SDK, no network call, ever.
- Clean, iOS-style UI, kept intentionally simple per v1 scope

## Setup

Requirements: Node ≥ 22.11 (RN 0.86 requires ≥ 20.19.4; the generated project pins ≥22.11),
a working RN CLI environment (Xcode for iOS, Android Studio/SDK for Android). If you don't
have that yet, follow "Set Up Your Environment" → "React Native CLI" at reactnative.dev first.

```bash
# 1. Install JS deps
npm install

# 2. iOS only — install native pods
cd ios && pod install && cd ..

# 3. Run
npx react-native run-ios
# or
npx react-native run-android
```

If you hit a Metro/babel cache issue after first install (common after adding Reanimated),
clear the cache once:
```bash
npx react-native start --reset-cache
```

### Why android/ and ios/ exist now
If you're comparing this to an earlier version of this project: the first draft only
included the JavaScript/TypeScript app code, not the native scaffolding. That's why
you couldn't find `android`/`ios` folders — they're generated by the RN CLI's own
project template (Gradle files, an Xcode project, a Podfile, etc.), not hand-written.
This version was regenerated with `npx @react-native-community/cli@latest init`, so the
native folders are real, then the app code was merged in on top.

### Notes on native modules
- `react-native-fs`, `react-native-share`, `react-native-document-picker` — autolink on RN
  0.86. If you're on an older RN version, check their READMEs for manual linking steps.
- `react-native-reanimated` v4 — needs `react-native-worklets` (already in `package.json`)
  and the babel plugin `'react-native-worklets/plugin'` (already in `babel.config.js`,
  must stay last in the plugins array). Requires a `pod install` on iOS.
- `react-native-gesture-handler` — `App.tsx` imports it at the very top, required.

## Project structure
```
App.tsx                        entry point, error boundary, store init
index.js, app.json             RN CLI entry registration (unchanged from template)
android/, ios/                 real native projects generated by RN CLI
src/
  types/habit.ts               Habit, HabitStats, ExportPayload, LogEntry types
  store/habitStore.ts          Zustand store: CRUD + streak/stat calculations
  services/
    storage.ts                 AsyncStorage read/write wrapper
    importExport.ts            JSON export via Share, import via DocumentPicker
    logger.ts                  offline error/event logger + global handler
    dateUtils.ts                minimal date helpers (no external date lib)
  components/
    HeatmapGrid.tsx             GitHub-style SVG contribution grid
    ProgressRing.tsx            animated 30-day completion ring (Reanimated)
    HabitCard.tsx                home list item
    ErrorBoundary.tsx
  screens/
    HomeScreen.tsx
    HabitDetailScreen.tsx
    AddEditHabitScreen.tsx
    SettingsScreen.tsx          export/import + diagnostics
  navigation/RootNavigator.tsx  bottom tabs + stack
```

## Not included in this v1 (by design, to keep scope/tokens tight)
- Custom reminder/notifications
- Weekly/custom frequency logic beyond the data field (currently all habits behave as
  daily; the `frequency` field is in the data model ready to extend)
- Themes/dark mode
- Widgets

These are straightforward to add on top of the current architecture — ask if you want any
of them built out next.
